Part 1: Identifying Usability Problems

Picking a Website

Website to Redesign: https://pcpartpicker.com/list/

Finding Problems

Usability: Is it easy to understand and use? (for all users)
  • Partpicker interface seems pretty straight forward
  • Users can browse for each part of a PC separately
Learnability: Is it easy to learn? (for new users)
  • For a new user building his or her first PC, the partpicker interface may be intimidating
  • Pictures and a short description of each part would make a more welcoming interface
Memorability: Is it easy to recall what to do? (for casual users)
  • Because the partpicker interface's usability is great due to its simplicity, memorability is also a strength of this interface
  • Pictures and short descriptions of each part will also help memorability along with learnability of the partpicker interface
Efficiency: Can typical tasks be done quickly? (for frequent users)
  • Partpicker interface's biggest weakness is how long it takes for users to browse for each part
  • All blue buttons link to a completely different page with a list of its corresponding PC part. Then, selecting a model will bring the user back to this page. A dropdown menu with some of the top rated models for each PC part will increase efficiency

Acessibility

The website did not display any major accessibility concerns. All text and interfaces such as buttons and links were easily visible and
able to be identified. VoiceOver on the MacOS also had no trouble reading all text on the PCPartpicker website line by line left to right.
